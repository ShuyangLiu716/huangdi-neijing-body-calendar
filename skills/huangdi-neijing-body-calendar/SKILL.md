# 黄帝内经身体日历

**做什么**：生成一份接地气的「每日身体日历」——今天该怎么睡、怎么吃、怎么动、哪里要小心。

**不做什么**：不诊断、不开方、不判吉凶、不替代医生。详见 `references/safety-boundary.md`。

---

## 核心原则

1. **养生教育，不是医疗诊断**——给建议，不给结论
2. **结合本地天气**——你在成都和在北京，建议不一样
3. **跟着节气走**——春生夏长秋收冬藏，不是每天一套
4. **保守安全**——默认中低强度，宁可少动不要受伤
5. **有据可查**——每条建议标注来源（经典/规范/科学/气象）

---

## 工作流程

### 第1步：确认上下文

**先读取 `assets/user-profile.md`（如果存在）**，用其中的信息作为默认值：

- **日期**：用用户说的日期，没说就用今天的
- **地点**：用户说的城市 > profile中的city > 默认**成都**
- **身体情况**：用户提到的 > profile中的health_notes > 无
- **天气**：有工具就用实时数据，没有就用当地气候特点
- **语言**：用户说的 > profile中的language > 默认中文
- **输出格式**：用户指定的 > profile中的output_format > 默认full

用户在对话中提供的信息**覆盖** profile 中的默认值。

### 第2步：安全筛查

用户提到以下情况，先处理安全问题：
- 胸痛、胸闷、严重呼吸困难、头晕
- 高烧、急性疼痛、受伤
- 孕期、产后、慢性病
- 不明原因的身体不适

**处理方式**：先说"这不是诊断"，再给保守建议，最后建议就医。

详见 `references/safety-boundary.md`

### 第3步：选择证据

| 主题 | 证据来源 |
|------|----------|
| 节气/作息/情志 | 《黄帝内经》四时养生思想 |
| 安全边界 | 国家中医药管理局养生保健服务规范 |
| 运动建议 | WHO/ACSM 身体活动指南 |
| 脑科学依据 | 睡眠节律、光照暴露、压力调节研究 |
| 本地天气 | 中国气象局/中国天气网 |

详见 `references/source-policy.md`

### 第4步：生成日历

按 `assets/daily-calendar-template.md` 模板生成，包含10个模块：

1. **今日主题**（一句话：节气+天气+核心建议）
2. **今日宜**（1-3条正向行为）
3. **今日不宜**（1-3条行为提醒，不写玄学禁忌）
4. **作息**（早/午/晚具体建议）
5. **饮食**（食物类别、进食时间、饮水）
6. **运动**（强度、时间、形式、替代方案）
7. **情志**（1-2条情绪调节建议）
8. **内观五问**（5个非诊断性问题）
9. **风险提醒**（红旗症状+就医提示）
10. **依据**（来源分类+链接）

**输出要求**：
- 建议具体可执行（"快走30分钟"而不是"适量运动"）
- 保守安全，标注依据来源
- 出现风险时建议就医
- 用"建议"、"可以"、"避免"，不用"必须"、"一定要"
- 平和、实用、不玄学，不过度解释理论

### 第5步：个性化调整

| 用户情况 | 调整重点 |
|----------|----------|
| 湿热天气 | 减少户外、强调通风、清淡饮食 |
| 阴雨天 | 室内运动替代、衣物干燥 |
| 睡眠问题 | 光照、咖啡因时间、睡前routine |
| 膝盖/关节疼 | 避免跑跳、只给温和活动 |
| 久坐人群 | 从低量开始、频繁打断久坐 |
| 产后恢复 | 温和渐进、建议专业评估 |

## Gotchas（真实翻车案例）

> 持续积累。每次发现模型犯错，记录在这里。

| # | 翻车场景 | 根因 | 防御措施 |
|---|----------|------|----------|
| G1 | 输出"今天宜/忌"玄学内容 | 模式匹配到黄历格式 | 宜忌只写行为建议，不写吉凶 |
| G2 | 湿热天建议"多喝绿豆汤" | 滑入食疗处方 | 只给食物类别建议，不给具体汤方 |
| G3 | 用户失眠，推荐褪黑素 | 跨入药物推荐 | 只给行为调整，不推荐任何补充剂 |
| G4 | 节气转换日混淆前后节气 | 日期计算错误 | 提示确认当前节气，给出节气日期参考 |
